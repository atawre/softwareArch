CS4015/SWE4403/CS6075 Design Pattern Assignment 3 

  

Due: Thu. 4:30pm, Feb. 14, 2018 

  

In this assignment, you will use Builder pattern and Singleton pattern to write the following programs.  

  

Create a program that uses the singleton pattern and the builder pattern to provide a suite of services: 

 

  • Word count: this service counts the number of words in a text file. 

  • Word table: this service returns a table containing all the words in a text file and the number of times they occur. For example, if the text file contains “Hello Hello this is what this hello is”, the resulting table will be as follows: 
     

  • Hello 

  • 3 

  • this 

  • 2 

  • is 

  • 2 

  • what 

  • 1 

  • Encryption: this service encrypts a text by matching every occurrence of a word with another word chosen from the input text using a 7-place substitution cypher. For example, if the file contains “one two three two four six one seven one eight nine”, and the substitution cypher is {1->5, 2->3, 3->1, 4->7, 5->2, 6->4, 7->6}, then the corresponding encryption table is: 
     

  • one, nine 

  • four 

  • two 

  • three 

  • three 

  • one 

  • four 

  • one 

  • six 

  • two 

  • seven 

  • two 

  • eight 

  • six 

Because one and two recur and thus are only counted once, and the resulting encrypted file is “four three one three one two four two four six four” because the substitution is based on the unchanged input file. If more than 7 unique words are in the file, they are replaced with the word at their position modulo 7, thus the encryption is (very) lossy. 
 

The program will ask for a file name and a service type, then use one of the services to create the output. Each of the services reads in the name of a text file and produces an output file of the same name with extension "Z1" for word count, “Z2” for word table, and “Z3” for encryption. Assume that there is no punctuation in the text file. Upper case letters should be converted into lower case. 

  

You should apply the Builder pattern in your programs to build each of the services, and apply the Singleton pattern to make sure that the builder inheritance hierarchy can only have a single instance (e.g., only one builder exist for each of the text services). 

  

  

Required development process for each of the program: 

  1. (Lab) In Rational Software Architect Designer, create a model project with name “CS4015DP3Model_StudentFullName” with a class UML diagram with name “SingletonBuilder”. 

  1. (Lab) In the class diagram, design classes and their relationships with as much as detail as possible. 

  1. (Lab) Generate Java code from the design classes in (2) into a Java project with name “CS4015DP3Java_StudentFullName”. 

  1. (Individual) Complete the programs by writing code for each method of each generated class. 

  1. (Individual) Test your program. 

Submission:  

  • Export the “CS4015DP3Model_StudentFullName” and “CS4015DP3Java_StudentFullName”projects to a single zip file with name “CS4015DP3_StudentFullName.zip”.  

  • Submit the zip file to the designated folder in the course D2L. 

 

Suggested hierarchy presented in lab is attached.
